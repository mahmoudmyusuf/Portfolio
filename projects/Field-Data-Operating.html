Project Overview

The Oil and Gas Field Data Analysis project focuses on extracting, processing, and analyzing industrial data collected from various sources within an oil and gas facility. The goal is to ensure accurate reporting, optimize operations, and maintain data integrity.

Industrial Data Flow from Field to Reporting: Oil and gas field transmitters are connected to the Distributed Control System (DCS), which collects data from multiple sources:

  • Direct Transmitter Readings: Pressure, Temperature, Level, Flow, Vibration, etc.
  • Local Control Packages: Gas Compressor Package, Air Dryer Package, Nitrogen Generation Package.
  • SCADA & Other Control Systems: Data from system-1 (Gas Compressor), Master Control Stations (sub-sea wells), and SCADA (Electrical Switchgear for motors & devices).
  • Flow Computers: Used for precise sales calculations by converting raw volumes to standard volumes.

This raw data can be:

  • Stored as received without conversion.
  • Converted into different units using simple or complex engineering formulas.
  • Summarized into daily totals for reporting.

All processed data is connected via EXA-OPC to PI-ProcessBook for real-time monitoring and reporting.

Data Cleaning & Preprocessing

  • Data Extraction: Using Excel PI Data Link add-in.
    • Tag Selection: Data is extracted based on predefined PI tags.
    • Time Settings:
      • Start Time: Defaults to current time.
      • End Time: Set based on historical data needs (daily, weekly, monthly).
      • Time Interval: Adjustable granularity (per minute, 5-minute, hourly).
  • Data Integrity Checks:
    • Freezing Check: Detects anomalies by comparing values over time.
    • Smoothing: Uses moving averages to reduce noise.
    • Incorrect Readings Removal: Filters out dead margins and cutoff values.
    • Unit Conversion: Applies appropriate conversion factors (e.g., Liter/min → BPD).

Data Integration Approach

After preprocessing, real-time transmitter data may contain deviations due to sensor errors, communication issues, or system downtimes. The following corrective measures are applied:

  • PI System Downtime: If data remains frozen for a certain period, a system notification is triggered.
  • Faulty Readings Validation:
    • Cross-verification using redundant transmitters.
    • Alternative validation methods, such as comparing level changes in storage tanks with flow readings.
    • Using flow computer data to confirm transmitter accuracy.
    • Historical DCS totalizers for allocation and adjustment.

Reporting & Monitoring

  • Uses available operating envelope data to compare actual vs. target values.
  • Triggers alerts or sends emails when values exceed operating limits.
  • Ensures real-time monitoring and historical analysis for decision-making.
