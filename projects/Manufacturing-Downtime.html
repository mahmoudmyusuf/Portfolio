üîÑ Data Cleaning & Preprocessing

The dataset consists of four primary tables and one data dictionary. The following steps were taken to ensure data quality:

  • Downtime Factors Sheet:
    • Loaded and selected the Downtime Factors sheet.
    • Promoted headers and changed data types appropriately.
    • Validated the data: No errors or empty records found.
  • Line Productivity Sheet:
    • Loaded and selected the Line Productivity sheet.
    • Promoted headers and changed data types.
    • Validated the data: No errors or empty records found.
    • Kept only required columns for analysis.
    • Calculated Actual Period (minutes):
      • Used for further calculations.
      • Ensured no negative values to confirm correct time entries.
  • Line Downtime Sheet:
    • Loaded and selected the Line Downtime sheet.
    • Promoted headers but found an extra undesired row.
    • Removed the top row before promoting headers.
    • Validated the data: No errors in the Batch column, but other columns had empty values.
    • Empty values were due to each factor being stored in separate columns.
    • Unpivoted 12 downtime factors into two columns: Factor and Downtime (minutes).
    • Revalidated the data: No errors or empty values in remaining columns.
  • Products Sheet:
    • Loaded and selected the Products sheet.
    • Promoted headers and changed data types.
    • Validated the data: No errors or empty records found.
    • Standardized Size column by converting all units to milliliters (ml).
    • Kept only relevant columns for analysis.

üîç Key Analysis Questions

The dataset was analyzed to answer critical operational and downtime-related questions:

  • Data Points Considered:
    • Downtime Factors (categorized by Operator Error: Yes/No).
    • Time Details (Date, Start, End, Actual Period).
    • Unique Identifiers (Batch, Product, Operator, Flavor).
  • Downtime Analysis (Sum, Average, Max, Min, Mode) based on:
    • Downtime Factor
    • Operator Error (Yes/No)
    • Operator Name
    • Batch
    • Product
    • Flavor

üìå Multi-Layered Analysis

The analysis was conducted in three structured layers:

  • Layer One:
    • Performed calculations without applying filters.
    • Derived aggregate statistics (sum, average, min, max, mode) for downtime.
  • Layer Two:
    • Separated calculations based on Operator Error (Yes/No).
    • Repeated the process for each:
      • Downtime Factor
      • Batch
      • Product
      • Operator
  • Layer Three:
    • Operator Error Downtime Factors:
      • Identified the most frequent downtime factor for each operator.
      • Insights:
        • If an operator repeatedly causes the same issue ‚Üí Needs Training.
        • If an operator contributes to multiple issues ‚Üí Needs Supervision.
        • If the same issue affects multiple operators ‚Üí Team Training Required (e.g., new technology or procedures).
    • Non-Operator Error Downtime Factors:
      • Identified the most frequent downtime factor for each product.
      • Insights:
        • If a product repeatedly faces the same issue ‚Üí New Instructions or Mentorship Required.
        • If the same issue affects multiple products ‚Üí New Technology or Upgrade Required.
